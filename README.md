## 🎯 项目简介

本项目通过**知识工程方法**，将14本经典投资著作的核心思想转化为结构化的**可执行决策技能（Decision Skills）**。每本书都被提炼为独立的决策手册，包含思维链示例（Thinking Chain）、选股决策树、买卖点判断规则、风险控制纪律等模块，帮助投资者将理论知识转化为实战能力。

### 核心特色

| 特色 | 说明 |
|------|------|
| 📖 **14本经典全覆盖** | 涵盖价值投资、趋势跟踪、技术分析、行为金融、交易心理五大流派 |
| 🧠 **思维链可视化** | 每个决策场景都有清晰的步骤化思考过程和怀疑点检查机制 |
| ⚡ **规则化表达** | 所有投资策略转化为"如果...就..."的可执行条件规则 |
| 🎯 **多流派兼容** | 每本书独立成Skill，按需调用，避免不同流派理论冲突 |
| 💼 **实战导向** | 统一的6+2模块结构，兼顾理论深度与实战可操作性 |
| 🤖 **AI原生设计** | 专为AI Agent/LLM优化，可直接作为System Prompt或知识库使用 |
| 📊 **超53万字** | 14个Skill文件总计约53万字，内容详实 |

## 💡 为什么做这个项目

### 投资者的痛点

```
❌ 读了很多投资书，但读完就忘，无法落地执行
❌ 不同流派的理论互相矛盾，不知道该听谁的
❌ 缺乏系统化的交易纪律，容易被情绪左右
❌ 投资决策时缺乏清晰的思维框架
❌ 想要系统学习但不知道从哪本书开始
```

### 本项目的解决方案

```
✅ 将每本书提炼为结构化决策手册 → 即查即用
✅ 每本书独立成Skill → 按需调用，不冲突
✅ 每个Skill都包含完整的风控和心态管理模块 → 纪律化
✅ 提供思维链示例 → 模拟专家思考过程
✅ 按流派分类，从入门到进阶 → 系统化学习路径
```

├── 📖 一、核心哲学与基本原则                ← 知识模块

│   └── 核心概念定义

│   ├── 第一层：大类筛选
│   ├── 第二层：行业/主题筛选
│   ├── 第三层：个股精选

├── 🛡️ 四、风险控制与心态纪律               ← 安全模块

## 🎬 使用场景示例

### 场景1：判断当前市场是否适合入场

**调用Skill**：《李大霄投资战略》

```
步骤1：估值水平评估
  → 当前A股整体PE是否低于历史均值？
  → 大盘蓝筹股PE是否低于10倍？
  → 股息率是否高于一年期银行存款利率？

步骤2：市场情绪判断
  → 周围人是否都在看空、远离股市？
  → 证券营业部是否门可罗雀？

步骤3：政策信号观察
  → 管理层是否密集出台利好政策？
  → 社保基金是否在悄然入市？

步骤4：历史点位对比
  → 对比7个历史底部：325/998/1664/1849/2440/2635/2689

步骤5：自我条件核查
  → 是否满足投资成功15个条件？

结论：5个条件满足4个 → 可以分批建仓蓝筹股
```

### 场景2：持仓出现亏损如何处理

**调用Skill**：《股票大作手回忆录》+《海龟交易法则》

```
《回忆录》提醒：
  "亏损是交易的成本，就像商人的进货成本一样"
  "不要和市场争辩，市场永远是对的"

《海龟法则》指令：
  → 检查是否触及2N止损位（入场价 - 2倍ATR）
  → 如果触及 → 立即离场，保护本金
  → 如果未触及 → 检查趋势是否仍然有效

决策：触及止损 → 立即离场 | 未触及 → 继续持有观察
```

### 场景3：短线追涨时机判断

**调用Skill**：《股票买卖定式》

```
检查1 - K线形态：是否出现突破缺口或放量大阳线？
检查2 - 均线系统：5日/10日/20日均线是否多头排列？
检查3 - 成交量：是否放量突破（成交量 > 前日2倍）？
检查4 - 技术指标：MACD是否金叉？KDJ是否未超买？

结论：4个条件满足3个 → 可以轻仓试多
      4个条件全部满足 → 可以标准仓位介入
      仅满足1-2个 → 继续观望
```

### 场景4：如何识别主力在出货

**调用Skill**：《股票买卖定式》主力操作分析

```
出货阶段特征检查：
  → 高位放量滞涨（成交量放大但价格不涨）
  → 频繁出现长上影线
  → 均线系统由多头转为空头排列
  → MACD顶背离（价格创新高但MACD不创新高）
  → 大单频繁卖出

决策：出现3个以上特征 → 逐步减仓或清仓
```

## 🛠️ 技术实现

### 知识工程流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  原始PDF  │ →  │ OCR提取  │ →  │ 内容     │ →  │ 规则化   │ →  │ 思维链   │ →  │  Skill   │
│  (扫描版) │    │(tesseract│    │ 结构化   │    │ 表达     │    │ 设计     │    │  生成    │
│           │    │+pdftoppm)│    │          │    │          │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 使用的工具

| 工具 | 用途 | 说明 |
|------|------|------|
| `pdftoppm` | PDF转图片 | 将扫描版PDF逐页转换为PNG图片 |
| `tesseract` | OCR文字识别 | 使用 `chi_sim+eng` 语言包识别中英文 |
| `TRAE SOLO` | AI知识工程 | 内容提取、结构化、规则化、思维链设计 |
| `Markdown` | 文件格式 | 统一的模板结构，便于版本管理和AI读取 |

### 核心挑战与解决方案

| 挑战 | 解决方案 |
|------|----------|
| 扫描版PDF无法直接提取文字 | 使用 `pdftoppm` + `tesseract` OCR技术批量识别 |
| 投资理念难以量化执行 | 设计思维链示例，将直觉判断转化为步骤化流程 |
| 不同流派理论互相冲突 | 每本书独立成Skill，用户按需选择调用 |
| 内容需兼顾深度与可读性 | 6+2模块结构平衡理论深度与实战指导 |
| 保持作者原汁原味的表达风格 | 语料库与金句库模块保留作者个性化语言 |
